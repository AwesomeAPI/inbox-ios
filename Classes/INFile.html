Overview

INAttachment represents a single attachment on an email message. Attachments may be created, uploaded and attached to drafts, or fetched from the server for existing messages.

resourceMapping

Subclasses override resourceMapping to define the mapping between their Objective-C @property’s and the key-value pairs in their JSON representations. Providing this mapping allows -initWithResourceDictionary: and resourceDictionary to convert the instance into JSON without additional glue code.

+ (NSMutableDictionary *)resourceMapping

Return Value

A dictionary mapping iOS property names to JSON fields.

Discussion

Subclasses override resourceMapping to define the mapping between their Objective-C @property’s and the key-value pairs in their JSON representations. Providing this mapping allows -initWithResourceDictionary: and resourceDictionary to convert the instance into JSON without additional glue code.

A typical subclass implementation looks like this:

  • (NSMutableDictionary *)resourceMapping { NSMutableDictionary * mapping = [super resourceMapping]; [mapping addEntriesFromDictionary: @{ @“firstName”: @“first_name”, @“lastName”: @“last_name”, @“email”: @“email” }]; return mapping; }

- (id)initWithFilename:(NSString *)filename mimetype:(NSString *)mimetype andData:(NSData *)data andPreview:(UIImage *)previewOrNil inNamespace:(INNamespace *)namespace

Parameters

mimetype

The mimetype of the attachment, such as “image/jpeg” or “text/plain”.

data

The attachment data.

previewOrNil

A small UIImage to use for displaying this attachment in your app. If you don’t provide a preview image, subsequent requests for this attachment’s localPreview may return nil.

namespace

The namespace to add this attachment in.

image

The display filename of the attachment.

Return Value

An initialized INAttachment object. To start uploading this attachment, you need to call upload.
