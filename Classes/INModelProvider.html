Overview

INModelProvider allows you to display, filter, sort, and refresh collections of Inbox objects without worrying about complexities of the underlying cache and API.

Providers expose a collection of items and a delegate protocol that allows you to update your interface when the collection of items changes.

Providers retrieve items from the local cache and also initiate requests to the Inbox API. When you set up a provider, you may receive several delegate calls as data is first retrieved from the cache and later merged with updated data provided by the API. It’s important to implement -provider:dataAltered: so your controller can react to incremental changes in the provider’s items array. In the future, the Inbox server will push data to your app, and -provider:dataAltered: may be called at any time.

Instance Methods

initWithClass:andNamespaceID:andUnderlyingPredicate:

Initialize a model provider for displaying a particular view of INModelObjects.

- (id)initWithClass:(Class)modelClass andNamespaceID:(NSString *)namespaceID andUnderlyingPredicate:(NSPredicate *)predicate

Parameters

modelClass

The class of item you want to display. Must be a subclass of INModelObject.

namespaceID

The namespace of items you want to display.

predicate

An optional predicate that will be used, in addition to the itemFilterPredicate, when querying the local cache and the Inbox API. This predicate cannot be changed after the provider is created, and is used by factory methods to return an instance of INModelProvider scoped to the messages in a particular thread, for example.

Return Value

An instance of INModelProvider
